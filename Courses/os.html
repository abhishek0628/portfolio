Basics of Operating Systems

1. What Is an Operating System (OS)?

An operating system is software that manages hardware resources and provides a platform for applications to run. It acts as a middleman between users and the machine, handling all the messy details so you don’t have to speak in binary.

2. Main Goals of an OS

  • Convenience: So users don’t need to write machine code just to open a file.
  • Efficiency: Ensures hardware resources are fully utilized.
  • Resource Management: Fairly distributes CPU time, memory, files, and I/O devices.
  • Security: Prevents processes from interfering with each other’s data.

3. Main Components

  • Kernel: The core that manages hardware, scheduling, memory, and devices.
  • Shell: The command-line interface or command interpreter (e.g., bash, PowerShell).
  • File System: Organizes and stores data.
  • Device Drivers: Translate OS commands into hardware operations.
  • User Interface: CLI or GUI—whatever keeps users from typing hex codes.

4. Key Concepts

  • Process: A running program (like Chrome devouring your RAM).
  • Thread: A lightweight unit of a process.
  • Scheduling: Determines which process gets CPU time next.
  • Memory Management: Allocates and frees up memory efficiently.
  • File Management: Handles data storage, retrieval, and protection.
  • I/O Management: Controls input and output devices such as keyboards and disks.

5. Types of Operating Systems

  • Batch OS: Executes jobs in batches with no user interaction.
  • Time-Sharing OS: Allows multiple users to share CPU time (e.g., Unix).
  • Real-Time OS: Used in systems where timing is critical (robots, medical devices).
  • Distributed OS: Runs across multiple interconnected systems.
  • Embedded OS: Powers smaller systems like routers, ATMs, and appliances.

Process Management in Operating Systems

1. What Is a Process?

A process is simply a program in execution. A program is just passive code stored on disk, but once the OS loads it into memory, assigns resources, and starts it running, it becomes a process.

Each process includes:

  • Code: The actual instructions to execute.
  • Program Counter (PC): Tracks the current point of execution.
  • Stack: Stores function calls and local variables.
  • Heap: Dynamic memory (from malloc or new).
  • Data Section: Contains global and static variables.

2. Process States

Processes change states constantly as they run. Typical states include:

  • New: The process is being created.
  • Ready: Waiting for CPU time.
  • Running: Currently executing on the CPU.
  • Waiting/Blocked: Paused, waiting for I/O or another event.
  • Terminated: Finished execution and released resources.

3. Process Control Block (PCB)

The OS maintains a Process Control Block for every process, storing all information needed to manage it:

  • Process ID (PID)
  • Current process state
  • CPU register values
  • Scheduling information
  • Memory management data
  • I/O status

When switching from one process to another, the OS saves the current PCB and restores another—this is a context switch, which enables multitasking (at the cost of speed).

4. Process Scheduling

The OS uses schedulers to decide which process runs next:

  • Long-term scheduler: Admits new jobs to the system.
  • Short-term scheduler: Selects the next ready process for CPU time.
  • Medium-term scheduler: Manages swapping processes between memory and disk.

5. Scheduling Algorithms

  • FCFS (First Come, First Served): Fair but can be inefficient.
  • SJF (Shortest Job First): Fast but impractical—job length isn’t always known.
  • RR (Round Robin): Each process gets equal CPU time; great for time-sharing systems.
  • Priority Scheduling: High-priority processes run first; may cause starvation.
  • Multilevel Queue: Groups processes into categories with different scheduling rules.

6. Inter-Process Communication (IPC)

Processes often need to exchange information. IPC provides mechanisms for safe communication:

  • Pipes: One-way communication between processes (e.g., parent ↔ child).
  • Message Queues: System-managed channels for sending messages.
  • Shared Memory: Fastest method; multiple processes share a common memory area.
  • Semaphores/Mutexes: Synchronization tools that prevent data conflicts.

7. Process Creation and Termination

  • Creation: Uses fork() in Unix or CreateProcess() in Windows.
  • Termination: A process may exit normally, crash (e.g., segmentation fault), or be killed by signals or user intervention.
