# 🔹 What is a Process?

A process is a program in execution, but that definition is almost insultingly incomplete.

A process is a complete execution environment, including:

- Code  
- Data  
- Execution state  
- Resources  

Think of it less like a file and more like a self-contained universe that the OS reluctantly keeps alive.

---

# 🔹 Process vs Program (people love confusing this)

## Program
- Passive (just a file on disk)
- Static
- Exists even when nothing is running it

## Process
- Active (running instance of a program)
- Dynamic
- Has memory, CPU state, and execution context

---

## Example

- One `.exe` file → can spawn multiple processes  
- Each process runs independently  
- Each gets its own memory space  

In short:  
A program is a recipe.  
A process is the cooking happening in your kitchen while everything is on fire and somehow still working.
